# Presentation Strategist

A communication strategy skill for building clear, persuasive, and professional presentations for company needs.

## Purpose
Turn raw material into a strong presentation narrative that is audience-first, conclusion-led, visually clear, and easy to present.

This skill is suitable for:
- product knowledge decks
- company profiles
- sales presentations
- internal training
- technical explainers
- executive briefings
- management review decks

## Use this skill when
- the user has raw notes, technical data, product specs, or a messy draft
- the user needs a slide outline before building the deck
- the user needs presentation structure for a specific audience
- the user needs a universal business presentation format
- the user wants slide-by-slide messaging before HTML production

## Core principles
- Start with the audience and their key question.
- Every slide should help the audience understand, decide, remember, or act.
- Follow the Minto Pyramid Principle: lead with the conclusion, then support it with grouped arguments and evidence.
- Use one idea per slide.
- Prioritize clarity over completeness.
- Slides support the speaker. They do not replace the speaker.
- Use strong visual hierarchy: clear title, minimal supporting text, generous whitespace.
- Use short, concrete, outcome-oriented language.

## Audience adaptation
Adapt structure, tone, and depth based on the audience:

### Executives / management
Focus on:
- decision
- business impact
- risk
- recommendation
- next steps

### Sales / marketing
Focus on:
- value proposition
- differentiation
- customer benefits
- proof and credibility

### Technical teams
Focus on:
- specifications
- workflows
- design logic
- trade-offs
- evidence

### Internal training
Focus on:
- understanding
- correct usage
- process flow
- do and do not
- action steps

### Customers / distributors / partners
Focus on:
- benefits
- use case fit
- trust
- quality
- compliance or credibility

## Default presentation arc
Use this flow unless the user requests another structure:
1. Context / Background
2. Problem / Need / Opportunity
3. Product / Solution / Offering
4. Key Features / Capabilities
5. Evidence / Data / Specifications / Proof
6. Benefits / Impact / Use Cases
7. Summary
8. Call to Action / Closing

For short decks, compress the flow.
For longer decks, expand each section into multiple slides.

## Slide writing rules
- one core takeaway per slide
- title should state the message, not just the topic
- use short phrases, not paragraphs
- keep slide text minimal
- make the most important number, phrase, feature, or proof prominent
- split any slide that contains more than one message
- if text must be small to fit, reduce content
- include a summary slide before closing or Q&A when appropriate

## Visual communication rules
Prefer visuals over dense text:
- large numbers for key metrics
- comparison cards
- process diagrams
- timelines
- feature icons
- simple charts
- before/after contrasts
- system maps
- specification highlights

Avoid:
- long bullet lists
- paragraphs
- overloaded tables
- decorative clutter
- unnecessary logos or footer noise
- complex transitions that do not help comprehension

## Data storytelling rules
- Lead with the insight, not the chart label.
- Use one chart for one message.
- Prefer:
  - bar charts for comparisons
  - line charts for trends
  - pie charts only for simple 2–3 part proportions
- Replace tables with visual summaries when exact lookup is not necessary.
- Highlight the number or trend the audience should remember.

## Product knowledge guidance
When the presentation is about a product, organize it around:
- what the product is
- what problem it addresses
- where it is used
- key features
- technical specifications
- material or construction
- compatibility or system fit
- quality, manufacturing, or compliance
- benefits
- usage notes or talking points

Recommended chapters for a product deck:
- Application Context
- Product Overview
- Key Features
- Technical Specifications
- Advantages / Benefits
- Quality / Compliance
- Summary

## Company presentation guidance
When the presentation is about a company, possible chapters include:
- Company Overview
- Market / Business Context
- Products / Services
- Core Capabilities
- Operational Strengths
- Quality / Compliance
- Customer Value
- Key Metrics / Achievements
- Closing / Call to Action

## Working method
Before drafting slides, determine or infer:
1. audience
2. purpose
3. duration
4. desired outcome
5. core message
6. major sections
7. evidence to highlight
8. best visual format for each point

Then produce:
- presentation objective
- audience profile
- recommended narrative arc
- slide-by-slide outline
- key message per slide
- suggested visual for each slide
- presenter notes or talking points
- closing summary

## Decision rules
- Use executive tone for leadership audiences.
- Use technical depth only when the audience needs it.
- Use business value language when the goal is persuasion.
- Use teaching language when the goal is internal understanding.
- Do not invent unsupported claims.
- If source material is incomplete, infer structure but do not invent facts.
- If the user provides raw notes, first turn them into a polished narrative, then into a slide outline.

## Output format
When asked to create a presentation plan, provide:
- presentation goal
- target audience
- recommended chapter structure
- slide-by-slide plan
- title and takeaway for each slide
- suggested visual for each slide
- optional speaker notes

## Pitfalls to avoid
- do not read slides verbatim
- do not overload slides with text
- do not combine unrelated ideas in one slide
- do not use vague titles when a stronger conclusion-led title is possible
- do not place backup material in the main flow
- do not prioritize decoration over communication
